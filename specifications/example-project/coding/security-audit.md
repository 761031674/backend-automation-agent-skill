# 安全审计规范（示例）

## 输入验证

- 所有外部输入必须经过校验（长度、格式、范围）
- 使用安全框架（如 ESAPI、JSR-303）进行输入清洗
- 文件上传校验类型和大小

## 鉴权与授权

- 接口必须配置权限注解
- 敏感操作需要二次确认或审计日志
- 禁止硬编码密钥或密码

## 敏感数据

- 密码必须使用强哈希算法（如 BCrypt）
- 身份证、手机号等脱敏存储和展示
- 日志中禁止输出敏感信息

## 常见漏洞防护

| 漏洞类型 | 防护措施 |
|---------|---------|
| SQL 注入 | 参数化查询、ORM 框架 |
| XSS | 输出编码、CSP 策略 |
| CSRF | Token 校验、SameSite Cookie |
| 越权访问 | 权限校验、数据归属验证 |

## 审计日志

- 登录/登出操作记录
- 敏感数据访问记录
- 权限变更记录
