# 代码审查规范（示例）

## 审查流程

1. **理解意图**：阅读 PR 描述，确认业务目标
2. **分层检查**：安全性 → 规范性 → 性能 → 可维护性
3. **输出报告**：分级标注（🔴阻塞项/🟡建议项/💭小改进）

## 检查清单

### 安全性（🔴）

- [ ] 无 SQL 注入（无字符串拼接 SQL）
- [ ] 无 XSS（输出已编码）
- [ ] 鉴权校验无绕过
- [ ] 安全框架验证正确使用

### 规范性（🔴）

- [ ] 实体类继承基础实体类
- [ ] 无 `@Data` 注解
- [ ] 使用 `@Resource` 注入
- [ ] SQL 无 `SELECT *` 且含逻辑删除条件

### 性能（🟡）

- [ ] 无 N+1 查询
- [ ] 循环内无数据库操作
- [ ] 大数据量分页合理

### 可维护性（💭）

- [ ] 命名清晰（类/方法/变量）
- [ ] 无魔法值，使用常量
- [ ] 复杂逻辑有注释

## 评论示例

```
🔴 [安全] UserController.java:42
SQL 注入风险：当前使用字符串拼接构建查询条件。
建议：使用参数化查询或 MyBatis `#{}` 占位符。
```
