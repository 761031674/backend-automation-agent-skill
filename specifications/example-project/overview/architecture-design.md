# 架构设计规范（示例）

## 架构原则

1. **单一职责**：每个服务只负责一个业务领域
2. **高内聚低耦合**：服务内部紧密协作，服务间通过明确接口通信
3. **可演进**：架构支持逐步拆分和扩展

## 服务拆分示例

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Gateway    │────→│  {svc-a}    │────→│  {svc-b}    │
│   网关层     │     │  业务服务A   │     │  业务服务B   │
└─────────────┘     └──────┬──────┘     └─────────────┘
                           │
                           ↓
                    ┌─────────────┐
                    │  {svc-c}    │
                    │  基础设施    │
                    └─────────────┘
```

## 通信方式

| 场景 | 通信方式 | 说明 |
|------|---------|------|
| 实时查询 | 同步调用（HTTP/RPC） | 低延迟、强一致 |
| 事件通知 | 异步消息（MQ） | 解耦、削峰填谷 |
| 定时任务 | 调度中心 | 批量处理、定时触发 |

## 领域建模示例

### 限界上下文

- **用户上下文**：用户注册、登录、权限管理
- **订单上下文**：订单创建、支付、履约
- **商品上下文**：商品信息、库存、价格
